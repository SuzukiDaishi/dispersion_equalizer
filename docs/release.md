# Release Guide

This project publishes builds through GitHub Actions.

Normal branch pushes and pull requests run CI and upload short-lived build
artifacts. Public GitHub Releases are created only when a `v*` tag is pushed.

## Release Checklist

1. Make sure the working tree is clean.

   ```powershell
   git status
   ```

2. Run the local checks.

   ```powershell
   cargo check
   cargo test
   cargo xtask bundle dispersion_equalizer --release
   ```

3. Update the version in `Cargo.toml` if needed, then commit the change.

   ```powershell
   git add Cargo.toml Cargo.lock
   git commit -m "Release v0.1.2"
   ```

4. Push the commit.

   ```powershell
   git push
   ```

5. Create and push a release tag.

   ```powershell
   git tag v0.1.2
   git push origin v0.1.2
   ```

6. Open GitHub Actions and wait for the `Release` workflow to finish.

7. Confirm the GitHub Release has these assets:

   ```text
   Dispersion.Equalizer-v0.1.2-windows-x64.zip
   Dispersion.Equalizer-v0.1.2-linux-x64.zip
   Dispersion.Equalizer-v0.1.2-macos-arm64.zip
   Dispersion.Equalizer-v0.1.2-macos-x64.zip
   ```

8. Download at least one zip and confirm the zip root contains:

   ```text
   Dispersion Equalizer.vst3
   Dispersion Equalizer.clap
   ```

## Notes

- Tags must use the `v*` format, such as `v0.1.2`.
- Release notes are generated by GitHub.
- Commit builds are stored as Actions artifacts for 7 days.
- Release builds target Windows x64, Linux x64, macOS arm64, and macOS x64.
- macOS builds are not signed or notarized yet.
- The local `package_release.ps1` script is only for local Windows packaging.
